# Lectura Analógica del TMP235 mediante ADC (Arduino)

Versión en Arduino de la práctica avanzada **Lectura analógica del TMP235 mediante ADC**, equivalente funcional del ejemplo en Pico SDK. Utiliza el core **arduino-pico** y las funciones estándar `analogRead()` / `analogReadResolution()`.

## Objetivo

Leer el voltaje de salida del sensor TMP235 mediante el pin GPIO26 (A0) del RP2040 y convertirlo a una lectura de temperatura en grados Celsius, aplicando la función de transferencia lineal del fabricante.

## Hardware

| Componente | Cantidad |
|---|---|
| Placa RP2040 compatible con Arduino | 1 |
| Sensor TMP235 (empaque SOT-23) | 1 |
| Cables de conexión | 3 |

## Conexiones

El TMP235 en empaque SOT-23 asigna los pines de la siguiente manera:

| Pin del TMP235 | Señal | Conexión en la placa |
|---|---|---|
| 1 | VDD | 3V3 |
| 2 | VOUT | GPIO26 (A0) |
| 3 | GND | GND |

> Si el sensor proviene de una placa breakout con pines serigrafiados, debe verificarse el orden ahí directamente, ya que no todas las breakouts respetan el orden crudo del empaque SOT-23.

## Función de transferencia

El TMP235 entrega un voltaje proporcional a la temperatura, válido en el rango de −40 °C a 100 °C:

```
Vout = 500 mV + 10 mV/°C × T
```

Despejando T:

```
T (°C) = (Vout_mV − 500) / 10
```

## Librerías requeridas

Ninguna adicional; se utilizan únicamente las funciones estándar del core **arduino-pico**.

## Código completo

```cpp
#define TMP235_PIN 26   // GPIO26 = A0 = ADC0 en el core arduino-pico

void setup() {
  Serial.begin(115200);
  analogReadResolution(12);   // El core entrega 10 bits por defecto; se fuerzan los 12 nativos
}

void loop() {
  int raw = analogRead(TMP235_PIN);              // 0 a 4095
  float voltage_mV = raw * 3300.0 / 4096.0;       // 3.3 V = 3300 mV
  float temp_c = (voltage_mV - 500.0) / 10.0;     // Función de transferencia

  Serial.print("raw=");
  Serial.print(raw);
  Serial.print("  V=");
  Serial.print(voltage_mV, 1);
  Serial.print(" mV  T=");
  Serial.print(temp_c, 2);
  Serial.println(" C");

  delay(500);
}
```

## Explicación

El core **arduino-pico** entrega, por defecto, valores de 10 bits (0-1023) en `analogRead()` para mantener compatibilidad con el Arduino clásico. La llamada `analogReadResolution(12)` en `setup()` fuerza el uso de la resolución nativa del ADC del RP2040 (0-4095), necesaria para que la fórmula de conversión coincida con la versión en Pico SDK. El pin puede referirse indistintamente como `26` o como `A0`, dado que el core mapea los pines Arduino directamente al número de GPIO físico.

## Verificación

Debe observarse en el monitor serie un valor de voltaje cercano a 500-800 mV a temperatura ambiente, correspondiente a una temperatura razonable (20-30 °C). Valores de voltaje cercanos a 3.3 V (o `raw` cercano a 4095) no son físicamente posibles para este sensor e indican un problema de cableado.

## Errores comunes

| Síntoma | Causa típica |
|---|---|
| Voltaje leído cercano a 3.3 V y temperatura fuera de rango físico (superior a 150 °C) | GND del sensor sin conexión común con la placa, o VDD/VOUT invertidos |
| `raw` limitado a 0-1023 en vez de 0-4095 | Falta la llamada a `analogReadResolution(12)` |
| `raw` constante en 0 | VOUT sin conexión, o cortocircuito a GND |
| Lectura ruidosa o inestable | Ausencia de capacitor de desacople (0.1 µF) entre VDD y GND del sensor |

## Variantes

- Sustituir la impresión por consola por una visualización en pantalla OLED (documentado por separado).
- Promediar N lecturas consecutivas para reducir el ruido de cuantización.
- Comparar la lectura contra `analogReadTemp()` (sensor de temperatura interno) para verificar consistencia relativa entre ambos.
